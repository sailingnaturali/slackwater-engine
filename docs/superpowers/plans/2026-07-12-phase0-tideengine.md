# Phase 0 — TideEngine Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A pure-Swift, offline harmonic tide/current engine (`TideEngine`) whose output matches the Neaps JS reference to ~1–2 cm / ~1 min, validated against official CHS (Victoria 07120) and NOAA (Friday Harbor) numbers.

**Architecture:** Port the Neaps *algorithm* (~600 lines: astronomy polynomials, Doodson V₀, Schureman/IHO node corrections, prediction sum, extremes finder) to Swift. Do **not** transcribe the ~5000-line constituent catalog — emit it as `catalog.json` from Neaps JS and load it as a bundled resource. Neaps JS (`@neaps/tide-predictor@0.10.0`) is the test oracle at every layer: astronomy → node corrections → per-constituent V₀/f/u → full prediction → extremes.

**Tech Stack:** Swift 6.3 (SwiftPM, macOS-runnable, no iOS dep for Phase 0), Swift Testing. Node 24 for the oracle/codegen scripts (`tools/*.mjs`).

## Global Constraints

- **License: MIT.** Public repo `slackwater-engine`. Header every source file accordingly.
- **No UI, no iOS-only APIs.** Engine is a plain SwiftPM library that runs headless on macOS (so `swift test` works in CI and locally).
- **Offline forever.** Node corrections/equilibrium args are computed on-device from astronomy for arbitrary future dates — never fetched, never precomputed-and-frozen.
- **Neaps is the oracle.** Every layer asserts against fixtures generated from `@neaps/tide-predictor@0.10.0`. Tolerance: heights ~0.02 m, times ~60 s, intermediate astro/f/u ~1e-6.
- **Faithful port.** Where a step says "port region X," reproduce the Neaps source math exactly; the golden test is what proves faithfulness. Neaps source regions (in `dist/index.js`): astronomy 1–180, node-corrections/iho 180–284, schureman 284–453, extremes 453–618, prediction 618–769, compound 769–948, constituent definition 948–1089, catalog 1089–6353.
- **Verify upstream license before publishing** (spec §3a): confirm Neaps permits an MIT Swift port. Non-blocking for building; blocking for `git push` public.

```
    └── gen-golden.mjs         # emit Tests/.../Fixtures/*.json (all layers)
```

Public API (`Station.swift`) — the surface every later phase and the app consume:

```swift
public struct HarmonicConstituent { public let name: String; public let amplitude: Double; public let phase: Double }  // phase = K (deg), amplitude in station units (m)
public struct TidePoint { public let time: Date; public let height: Double }
public enum ExtremeKind { case high, low }
public struct TideExtreme { public let time: Date; public let height: Double; public let kind: ExtremeKind }

public struct Station {
    public init(constituents: [HarmonicConstituent], offset: Double = 0)
    public func heights(from: Date, to: Date, step: TimeInterval) -> [TidePoint]
    public func extremes(from: Date, to: Date) -> [TideExtreme]
}
```
